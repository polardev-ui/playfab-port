Overview

PlayFab Port is an open-source backend wrapper that makes migrating from PlayFab to Supabase seamless for game developers.

What is PlayFab Port?

PlayFab Port provides a C# wrapper around Supabase, replicating PlayFab's functionality while leveraging the power of PostgreSQL, Row Level Security, and Edge Functions. It's designed for indie game developers who need a scalable, affordable backend.

Why Migrate?

PlayFab's free tier was recently reduced from 100,000 users to just 1,000 users. This dramatic cutback makes it impossible for indie developers to scale affordably. Supabase offers:

500GB of database storage on the free tier
Unlimited users without scaling costs
Open-source PostgreSQL with full control
No vendor lock-in – self-host if needed
Row Level Security for built-in data protection

Key Features

Device-based authentication using SystemInfo.deviceUniqueIdentifier
Singleton SupabaseManager for easy integration
Modular architecture for Gorilla Tag and general games
Real-time data subscriptions via WebSocket
Edge Functions for secure server-side logic
Relational SQL schemas with built-in RLS policies

Installation

Get PlayFab Port up and running in your Unity project.

Prerequisites

Unity 2020.3 LTS or newer
A Supabase project (free tier at supabase.com)
Supabase C# SDK installed in your project

Step 1: Clone the Repository

git clone https://github.com/polardev-ui/playfab-port.git
cd playfab-port

Step 2: Copy Assets to Your Project

Copy the Assets folder from PlayFab Port into your Unity project's Assets folder:

Assets/SupabaseWrapper/ ├── Core/ ├── Copy/ └── General/

Step 3: Install Supabase-C#

Via NuGet (recommended for Unity Package Manager compatibility):

dotnet add package Supabase.Core

Step 4: Configure Your Project

Create a new GameObject in your loading scene and attach the SupabaseManager component. In the inspector, paste your Supabase URL and Anon Key.

Step 5: Run the Schema

In your Supabase dashboard, navigate to the SQL Editor and run the appropriate schema file:

schemas/general/schema.sql
or
schemas/copy/schema.sql

Architecture

Understand how PlayFab Port is structured and how components interact.

System Design

PlayFab Port follows a three-layer architecture:

Layer 1: Core Engine

The SupabaseManager singleton initializes and manages your Supabase connection. It handles:

Session initialization and validation
Authentication state management
Real-time subscription setup

Layer 2: Security Layer

PostgreSQL Row Level Security (RLS) policies ensure data isolation:

Each user can only access their own data
Sensitive operations trigger via the Anon Key safely
Server-side Edge Functions handle privileged actions

Layer 3: Application Layer

Choose your implementation path based on your game type:

Copy (Gorilla Tag Replicas)

Device ID authentication, cosmetics tracking, room management, and player metadata tailored for social VR games.

General (Standard Games)

Email/password authentication, player profiles, XP/level systems, currency management, and leaderboards for all game types.

Data Flow

Client (Unity)

↓ Auth Request ↓

SupabaseManager (Singleton)

↓ Send to Supabase ↓

Supabase (Auth + Database)

↓ RLS validates request ↓

PostgreSQL Database

↓ Returns encrypted token ↓

Client (Unity) – Now authenticated

Authentication

Learn how to authenticate players in your game using PlayFab Port.

Authentication Methods

PlayFab Port supports two primary authentication methods depending on your game type.

Device ID Authentication (Copy)

Used for Gorilla Tag replicas and similar VR games.

GorillaAuth auth = GetComponent<GorillaAuth>();
await auth.AuthenticateGorilla();

// Automatically uses SystemInfo.deviceUniqueIdentifier // No email or password required

Email/Password Authentication (General)

Used for traditional games with player accounts.

Sign Up

GeneralAuth auth = GetComponent<GeneralAuth>();
await auth.SignUp("player@game.com", "SecurePassword123", "PlayerName");

Sign In

GeneralAuth auth = GetComponent<GeneralAuth>();
await auth.SignIn("player@game.com", "SecurePassword123");

Session Management

// Check if user is authenticated
bool isAuthenticated = auth.IsAuthenticated();

// Get current user ID
string userId = auth.GetCurrentUserId();

// Refresh token
await auth.RefreshToken();

Best Practices

Always store sessions securely using PlayerPrefs encryption
Implement token refresh before expiration (auto-handled by wrapper)
Never expose your Service Key – use Anon Key only
Validate sensitive actions on the server via Edge Functions

Database

Understand the database schema and how to work with data.

Schema Overview

PlayFab Port provides pre-built SQL schemas optimized for game backends.

General Schema Tables

profiles

Stores player profile information including username, displayname, and metadata.

player_stats

Tracks XP, level, and currency (gold, gems, etc.) with Row Level Security.

inventory

Manages player items with quantities and metadata. RLS ensures players only see their own inventory.

leaderboards

Read-only materialized view for competitive rankings. Updated via Edge Functions.

Copy Schema Tables

players

Gorilla Tag player data with device ID, cosmetics, and social info.

cosmetics

Cosmetic tracking for hats, hand colors, and avatar customization.

rooms

Room metadata and player presence tracking.

Row Level Security

All tables include RLS policies that automatically enforce data isolation:

-- Example RLS policy (auto-generated)
CREATE POLICY player_own_data ON profiles
FOR ALL
TO authenticated
USING (auth.uid() = user_id)
WITH CHECK (auth.uid() = user_id);

Querying Data

// Get current player's stats
var stats = await GetComponent<SupabaseManager>().from("player_stats").select().single();

API Reference

Complete reference for PlayFab Port classes and methods.

SupabaseManager

The main singleton that manages your Supabase connection.

Methods

Method Description
Initialize(url, anonKey) Initialize Supabase connection
IsAuthenticated() Check if user is logged in

Authentication for standard games.

Methods

Method Description

Methods

Security

Best practices and security considerations for using PlayFab Port.

Core Security Principles

Never Expose Service Keys

Always use the Anon Key in your client application. The Service Key should only ever be used in server-side code or Edge Functions.

Row Level Security

RLS policies are enabled by default on all tables. Players can only access their own data through the Anon Key.

Server-Side Validation

Critical operations like currency transfers, item grants, or stat modifications must run on the server via Edge Functions.

Data Protection

Encryption in Transit

All data is transmitted via HTTPS. Supabase uses TLS 1.2+ by default.

Authentication Tokens

JWT tokens are short-lived. The wrapper automatically refreshes them before expiration.

Session Management

Sessions are stored locally with encryption. Clear sessions on logout and when switching devices.

Common Vulnerabilities to Avoid

Do not modify user ID in requests on the client
Do not trust client-side stat calculations – validate on server
Do not store sensitive data unencrypted in PlayerPrefs
Do not disable RLS policies for convenience

Auditing and Monitoring

Enable Supabase audit logs in your dashboard to track all database changes. Monitor for suspicious patterns like:

Bulk stat modifications
Unusual query patterns
Failed authentication attempts

Examples

Learn through practical code examples.

Complete Game Initialization

public class GameManager : MonoBehaviour {
    private SupabaseManager supabaseManager;
    private GeneralAuth auth;

    void Start() {
        supabaseManager = GetComponent<SupabaseManager>();
        auth = GetComponent<GeneralAuth>();
        
        if (supabaseManager.IsAuthenticated()) {
            LoadPlayerData();
        } else {
            ShowLoginUI();
        }
    }
}

Fetching Player Stats

async void LoadPlayerData() {
    try {
        var response = await supabaseManager
            .from("player_stats")
            .select()
            .single();
        
        var stats = JsonConvert.DeserializeObject<PlayerStats>(response);
        DisplayStats(stats);
    } catch (Exception e) {
        Debug.LogError("Failed to load stats: " + e.Message);
    }
}

Real-Time Leaderboard Updates

void SubscribeToLeaderboard() {
    var subscription = supabaseManager
        .from("leaderboards")
        .on(RealtimeEventType.All, OnLeaderboardChange)
        .Subscribe();
}

void OnLeaderboardChange(RealtimePayload payload) {
    // Update UI with new leaderboard state
    RefreshLeaderboardDisplay();
}
